## Why oidc-exchange?

<CardGrid stagger>
  <Card title="No vendor lock-in" icon="open-book">
    Your user data stays in your database, your tokens are signed with your keys, and your configuration is a TOML file in your repo.
  </Card>
  <Card title="Single-purpose" icon="approve-check-circle">
    Validate upstream identity, issue downstream tokens. No login UIs, no session management, no consent screens.
  </Card>
  <Card title="Pluggable architecture" icon="setting">
    Hexagonal design with trait interfaces. Swap DynamoDB for PostgreSQL or KMS for a local key — implement a trait, not a rewrite.
  </Card>
  <Card title="Deploy anywhere" icon="rocket">
    Same binary runs as an AWS Lambda function or a long-lived axum server. Deploy to Fargate, a Linux VM, or a Kubernetes cluster.
  </Card>
</CardGrid>

## Supported backends

| Component | Adapters |
|-----------|----------|
| **User storage** | DynamoDB, PostgreSQL, SQLite |
| **Session storage** | DynamoDB, PostgreSQL, SQLite, Valkey/Redis, LMDB |
| **Key management** | AWS KMS, local Ed25519/ECDSA files |
| **Audit trail** | CloudTrail Lake, SQS (to S3/Iceberg), noop |
| **Identity providers** | Google, Apple, any standard OIDC provider |
| **Telemetry** | OTLP, X-Ray, stdout |
